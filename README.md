# To‑Do List App (Next.js + Supabase)

This is a minimal to‑do list built with Next.js and Supabase, ready to deploy to Vercel.

Features:
- Add, toggle (done), and delete tasks
- Uses Supabase `todos` table for persistence

Getting started (local):

1. Create a Supabase project and create a `todos` table with this SQL:

```sql
create table todos (
  id bigint generated by default as identity primary key,
  content text not null,
  done boolean default false,
  inserted_at timestamptz default now()
);
```

2. Copy `.env.example` to `.env.local` and set your values:

```
NEXT_PUBLIC_SUPABASE_URL=https://your-project-ref.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
```

3. Install and run:

```bash
npm install
npm run dev
```

Open `http://localhost:3000`.

Deploying to Vercel:

- Push this repo to GitHub and import it into Vercel.
- In the Vercel project settings, add the following Environment Variables (Production & Preview):
  - `NEXT_PUBLIC_SUPABASE_URL` = your Supabase URL
  - `NEXT_PUBLIC_SUPABASE_ANON_KEY` = your anon/public key
- Deploy — Vercel will run `npm run build` automatically.

Notes:
- This example uses the client-side anon key. For private operations, add server-side API routes and service_role key (keep it secret).
- You can enable Realtime or Row Level Security policies in Supabase as needed.
